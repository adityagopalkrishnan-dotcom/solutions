1. What problem does this solve?
QuestionPro CX gives teams the data. The Dashboard Chef gives them the signal.
Out-of-the-box QuestionPro dashboards show gauges, bars, and numbers — but they don't explain what the numbers mean, what changed after a filter was applied, or what to do next. For organisations with a mixed audience — executives who want one number, managers who want trends, analysts who want distributions — a single static layout serves none of them well.
Custom CX Dashboard Widgets replace the native QuestionPro tiles with interactive, audience-aware visualisations that read the same live data, respond to the same filters, and surface plain-English insights alongside the charts — including AI-generated action items from the QuestionPro AI Router.
2. What is the solution and how does it work?
What it does. JavaScript widgets injected into QuestionPro's Custom HTML Widget tile read live data from native QuestionPro source widgets on the same dashboard, render a bespoke UI, and re-render automatically when any dashboard filter is applied. No page reload. No data export. No separate tool.
How it fits the product. The entire solution runs inside the QuestionPro CX dashboard the customer already uses. Custom widgets sit on top of native QuestionPro widgets, reading from them and hiding them. The native widgets keep doing their job — receiving data from QuestionPro's backend on every filter change — while the custom widgets handle the visual layer.

Filters update every widget simultaneously — and the AI Insights widget re-generates its recommendations based on the filtered view automatically.

7. How do you demo it?
Recommended demo flow
Open the dashboard — show the clean layout with no white boxes or raw native charts visible. Point out the five custom widgets.
Click the Top Box metric card on the CSAT Overview — show the year-by-year trend panel expanding inline. Click CX Health to show the CSAT 50% + FCR 30% + CES 20% score breakdown.
Apply a dashboard filter (e.g. Support Channel: Chat) — show all four data widgets updating simultaneously within 1–2 seconds.

Point at the AI Insights widget — hit Refresh and walk through the headline, top three actions, and forecast risk statement generated from the current filtered data.

9. Competitive angle
Other BI tools (Tableau, Power BI, Looker) can build interactive dashboards from QuestionPro data — but they require a data extract, a separate licence, and a separate login. This solution makes the QuestionPro dashboard itself the interactive tool, keeping the team in one place.
QuestionPro's own AI features generate insights at the question level. The Dashboard Chef aggregates all widget data — CSAT, effort, resolution, segments — into a single AI prompt that produces cross-metric insights and prioritised actions, not per-question breakdowns.
Custom dashboard vendors typically require a separate SaaS tool. This pattern requires only the QuestionPro CX licence the customer already has, plus a services engagement to set it up.
Because the widgets are JavaScript running inside QuestionPro, they respond to the same filters, date ranges, and segments as every other dashboard widget — no separate filter logic to maintain.
Sales positioning. Lead with what they already have: "You're already paying for QuestionPro CX and collecting the data. We're not asking you to add another tool or another login. Component
Role in the solution
Source widget (QP native)
The real QuestionPro widget — gauge, tracker, cross-tab, trend comparison. Kept on the dashboard and hidden from users. QuestionPro updates it on every filter change.
Custom HTML Widget (target)
The blank QuestionPro tile where the JavaScript renders the custom UI. This is what the user sees.
JavaScript widget script
Self-contained script pasted into the Custom HTML Widget editor. Reads data, renders UI, detects filter changes, and re-renders automatically.
findChart() pattern
The function that reads live Highcharts data from source widgets, skipping stale destroyed instances correctly.
MutationObserver + poll loop
Detects filter changes and triggers re-render. Observer catches DOM changes; 750ms poll catches anything the observer misses.
QuestionPro AI Router
Called by the AI Insights widget to generate plain-English insights, prioritised actions, and forecast risk from live dashboard data.
User applies filter  →  QP updates source widget  →  Observer / poll detects change  →  Custom widget reads new data  →  Custom widget re-renders  →  User sees updated signal  →  AI Insights auto-refreshes with new context

If the customer says…
Your response
"We can build this ourselves — we have developers."
You can — and we can hand you the architecture. But the hard part isn't the HTML. It's knowing exactly how QuestionPro's Highcharts API works, which hiding method causes React to unmount the widget, and how to detect filter changes without an event hook. That took weeks of live debugging. We give you the working result, not the learning curve.
"Does this break when QuestionPro updates?"
The widgets read from Highcharts series data and DOM table structures — both stable, documented interfaces. They do not rely on internal QuestionPro API calls or undocumented endpoints. We also offer a maintenance tier that monitors and updates scripts if anything changes.
"We already use Power BI / Tableau for our dashboards."
Great — those are excellent analysis tools. This is different: it makes the QuestionPro dashboard itself the presentation layer, so your CX team gets live, filter-responsive insights without exporting data or switching tools mid-meeting.
"Is the AI trustworthy? What if it says something wrong?"
The AI Insights widget shows exactly what data it used (response count, CSAT %, CES mean, segment scores) in the metric pills at the top. The insights are grounded in the live dashboard data — not generated from general knowledge. If the data is right, the insights are directionally right.
"Can we customise what the widgets show?"
Yes. Each widget has configuration variables at the top of the script (benchmark threshold, colour scheme, segment labels). More significant changes — different chart types, different KPIs — can be generated from the LLM template in the solution cookbook.
A dashboard that shows charts is a reporting tool. A dashboard that surfaces signals, explains what changed, and tells you what to do next is a decision tool. That is the difference The Dashboard Chef makes.

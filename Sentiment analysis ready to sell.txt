1. What problem does this feature solve?

Open-ended survey responses contain rich insight, but they are unstructured, inconsistent and difficult to analyze at scale. Most organizations either review comments manually or push the data into separate analytics tools that do not integrate cleanly with core CX dashboards.

AI Router solves this by automatically converting qualitative responses into structured, actionable data - including sentiment, themes and summaries - directly within the survey and respondent dataset.

2. What is the feature and how does it work?

What it does

AI Router processes open-ended responses in real time and transforms them into structured CX intelligence. It can extract sentiment, identify key themes, classify topics and generate respondent-level executive summaries mapped into predefined fields.

How it fits into the existing product

AI Router is fully embedded within QuestionPro CX surveys. Responses are sent to AI Router through a webhook or router call, and the returned JSON output is stored as custom variables within the same survey dataset. This makes the data immediately usable in dashboards, filters, reporting views, workflows and raw data exports.

Core architecture

  --------------------------------------------------------------------------------------------------------------------------
  Component                     Role in the solution
  ----------------------------- --------------------------------------------------------------------------------------------
  Survey open-ended questions   Capture the raw qualitative feedback from customers, employees or respondents.

  Webhook / AI Router call      Sends each open text response or combined respondent payload to AI Router in real time.

  Prompt and codebook           Defines the classification rules, accepted themes, sentiment scale and summary format.

  Structured JSON output        Returns normalized fields such as sentiment, theme, subtopic and executive summary.

  Custom variables              Store AI-generated values in the same respondent record as the original response data.

  CX dashboards and exports     Use router outputs as filters, widget dimensions, reporting fields and exportable columns.
  --------------------------------------------------------------------------------------------------------------------------

Core flow

  --------------------------------------------------------------------------------------------------------------------------------------------
  Stage                          How it works
  ------------------------------ -------------------------------------------------------------------------------------------------------------
  Response capture               Open-ended responses are collected within the survey experience.

  AI processing                  Responses are sent to AI Router through a webhook or router configuration.

  Structured output generation   AI Router returns a controlled JSON object with sentiment, theme, summary and any additional agreed fields.

  Custom variable mapping        Each JSON field is mapped into a defined custom variable so it stays within the respondent record.

  Dashboard integration          The AI fields become filterable dimensions, visual widgets, exportable columns or workflow triggers.

  Operational action             CX, HR, research or operations teams use the structured insight to identify drivers and prioritize fixes.
  --------------------------------------------------------------------------------------------------------------------------------------------

Structured output model

The exact output schema can be customized by use case, but the recommended baseline model should keep the output controlled, concise and dashboard-friendly.

  -----------------------------------------------------------------------------------------------------------------------------------------
  Output field                What it captures
  --------------------------- -------------------------------------------------------------------------------------------------------------
  sentiment                   Overall sentiment of the comment or response, using an agreed scale such as Negative, Neutral and Positive.

  theme                       The main topic or experience driver identified in the feedback.

  subtopic                    A more specific category beneath the primary theme when needed.

  summary                     A short respondent-level or comment-level summary written for executive readability.

  quality / confidence flag   Optional flag for low-signal, unclear, irrelevant or incomplete responses.

  recommended action          Optional operational signal that can guide follow-up, escalation or improvement work.
  -----------------------------------------------------------------------------------------------------------------------------------------

When/how a customer would use it

Customers use AI Router when they need to analyze large volumes of qualitative feedback at scale while keeping insight directly embedded inside QuestionPro CX. It is strongest when open-ended comments must feed dashboards, filters, reporting and action instead of remaining as raw text in exports.

Typical examples include employee experience surveys, CX open-ended feedback analysis, Voice-of-Employee programs, Voice-of-Customer programs, post-interaction comments, product feedback and qualitative insight reporting.

Great fit vs. poor fit

  -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  Strong fit                                                                                                   Poor fit / proceed with caution
  ------------------------------------------------------------------------------------------------------------ ------------------------------------------------------------------------------------------------------------
  Surveys carry multiple open-ended questions and the team wants them analysed consistently at scale.          Only one or two free-text fields with low volume — manual review is genuinely cheaper than configuration.

  The customer wants qualitative insight living inside their existing CX/XA dashboards, not a separate tool.   The customer has standardised on a dedicated text-analytics platform and has no appetite to move analysis.

  They have, or will define, a theme taxonomy / codebook for their context.                                    No agreed taxonomy and no stakeholder willing to own one — output feels generic and trust drops.

  Open-ended responses need to drive filters, widgets, exports or workflows.                                   Comments are collected but never acted on — no downstream use to justify the build.

  Response volume is high enough that consistency and speed matter.                                            Very low volume where a human can read every verbatim in minutes.

  “Why not just have our analysts code it?”                Manual coding is slow, inconsistent across coders, and does not scale across regions or waves. This keeps classification consistent and live, freeing analysts to act rather than tag.

6. How do you demo it?

Recommended internal demo flow:

  1. Start with a traditional survey that contains one or more open-ended questions.

  2. Show the challenge: raw comments are useful but hard to summarize, segment or filter consistently.

  3. Introduce AI Router processing in real time through the survey/webhook flow.

  4. Submit example comments and show the structured output fields such as sentiment, theme and summary.

  5. Show where the JSON outputs are stored as custom variables in the respondent record.

  6. Open the CX dashboard and demonstrate how AI fields become filters, widgets or segments.

  7. Show a respondent-level executive summary to make the output useful for leadership review.

  8. Close with the operational value: faster insight, less manual coding and no separate analytics silo.

10. FAQ

Does this require a separate analytics platform?

  No. Outputs are embedded within the survey and available in the same CX dashboards, exports and respondent dataset.

Can this handle multiple open-ended questions?

  Yes. Each response can be processed individually, and the results can be combined into respondent-level summaries when needed.

Is the output customizable?

  Yes. Prompts can be tailored to return specific themes, classifications, summaries, confidence flags or action signals based on the customer’s business needs.

Does this work in real time?

  Yes. Responses can be processed as they are submitted, with outputs available immediately for downstream reporting or workflows.

Can this data be exported?

  Yes. AI-generated fields are stored as custom variables and can be included in raw data exports alongside standard survey responses.

What should be scoped before selling it?

  Confirm the number of open-ended fields, expected response volume, theme taxonomy, required output fields, dashboard needs, language requirements and whether respondent-level summaries are needed.
